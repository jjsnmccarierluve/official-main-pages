# official-main-pages

本仓库用于归档和发布多个独立 HTML 页面。每个页面均为自包含的静态资源，不依赖外部域名或特定后端服务。

## 项目简介

这是一个通用的 HTML 页面归档仓库。仓库内的页面可被直接访问或嵌入，适用于需要集中管理多个静态展示页面的场景。所有页面均为独立文件，彼此无耦合关系。

## 目录说明

```
├── README.md          # 本文件
├── index.html         # 仓库索引页（可选）
├── pages/             # 页面归档目录
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
└── assets/            # 公共资源（CSS/JS/图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：存放所有归档的 HTML 页面，文件名应具有辨识度。
- `assets/`：存放页面共享的静态资源，避免重复引入。
- `index.html`（可选）：若需要，可提供一个简单的导航页。

## 页面归档说明

- 每个页面应保持独立，不引用仓库外的资源。
- 页面可引用 `assets/` 目录下的本地资源。
- 建议在页面头部添加简要说明或版本注释，便于后续维护。
- 归档的页面不应包含任何跟踪代码、外部请求或动态内容。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，必要时更新 `index.html` 中的链接。
- 修改页面：直接编辑对应文件，注意保持向后兼容。
- 删除页面：移除文件并清理相关引用。
- 资源更新：修改 `assets/` 下文件时，需同步更新所有引用的页面。
